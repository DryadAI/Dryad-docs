## What the School of Memory Does

The School of Memory is responsible for the persistence and promotion of information within the Dryad system. It manages the transition of data from volatile execution traces to permanent semantic and procedural baselines.

## Components

### Infrastructure Pillar
- **Redis Audit Streams:** Live trail of system activity.
- **Qdrant Vector DB:** Semantic search and long-term memory store.
- **Mem0:** Personal and procedural memory management.

### Communication Pillar
- **Trace Promotion Protocol:** Rules for what becomes a "fact".
- **Semantic Anchor Scheme:** `dryad-docs://` URI resolution.

### Philosophy Pillar
- **Identity Mirroring:** Ensuring the system's "soul" is mirrored across all components.
- **Persistence-as-Integrity:** Accurate recording of failure and success.
